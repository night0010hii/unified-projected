# Secure File Transfer Monitoring System

A Python-based blue team desktop GUI tool to monitor file transfers in real time,
detect unauthorized data movement, verify file integrity via SHA256 hashing,
and generate full audit reports.


## Where Output Is Stored

Everything the system produces is saved inside the `logs/` folder:

| File | What it stores | Created by |

| `logs/file_transfer_log.json` | Every file event with timestamp, path, hash, sensitivity flag | `monitor.py` / `demo_data.py` |
| `logs/alerts.json` | All security alerts with type, severity, message | `alert_engine.py` / `demo_data.py` |
| `logs/hash_db.json` | SHA256 baseline — one entry per file | `integrity_checker.py` / GUI baseline button |
| `logs/integrity_report.json` | Last integrity scan — OK / TAMPERED / NEW / DELETED | `integrity_checker.py` |
| `logs/audit_report.txt` | Human-readable full report | GUI Export Report button |
| `demo_files/` | All test files generated by `demo_data.py` | `demo_data.py` |

## GUI Tabs Explained

### Live Feed

Real-time table of all file events. Color coded:

- Yellow row — sensitive file type (.pdf .csv .sql .env .docx etc.)
- Red row — deleted or tampered sensitive file
- Green row — new sensitive file created
- Gray row — normal non-sensitive file

Filter buttons at the top: ALL  CREATED  MODIFIED  MOVED  DELETED
Double-click any row to see full event details.

### Alerts

All security alerts with severity levels:

| Alert Type | Severity | Trigger |

| INTEGRITY | CRITICAL | File hash changed after baseline |
| UNAUTHORIZED | CRITICAL | Sensitive file moved to suspicious location |
| SENSITIVE_FILE_DELETED | HIGH | Sensitive file was deleted |
| BULK_TRANSFER | HIGH | 50+ file events in 60 seconds |

Click Acknowledge All to clear the unread count.

### Integrity

Two buttons:

- Take Baseline Snapshot — hashes all files now, saves to logs/hash_db.json
- Run Integrity Check — compares current files against baseline, shows:
  - OK — file unchanged
  - TAMPERED — hash mismatch, file was modified
  - NEW — file not in baseline
  - DELETED — file in baseline but no longer on disk

### Log Explorer

Full searchable history. Search box filters by filename, path, or hash in real time.
Click Refresh to reload from disk.

## Testing with Demo Data

```bash
python demo_data.py
```

This script automatically:

1. Creates 5 normal files in demo_files/safe/
2. Creates 6 sensitive files in demo_files/sensitive/
3. Takes a SHA256 baseline snapshot of all files
4. Tampers with employees.csv and api_keys.env  →  triggers INTEGRITY alerts
5. Moves financial_report.xlsx to outbox/  →  triggers UNAUTHORIZED alert
6. Deletes contract_2024.pdf  →  triggers SENSITIVE_FILE_DELETED alert
7. Creates 60 CSV files rapidly  →  triggers BULK_TRANSFER alert
8. Adds 10 extra varied events for Log Explorer testing

Expected GUI stats after running demo_data.py:

| Stat card | Value |

| Total Events | 85 |
| Alerts | 5 |
| Sensitive Files | 76 |
| Integrity Fails | 2 |

After running demo, open the GUI and:

1. Sidebar → + Add Dir → select the demo_files/ folder
2. Click Verify Integrity → see employees.csv and api_keys.env as TAMPERED
3. Check Alerts tab → 5 red alerts waiting
4. In Log Explorer search csv → filters all CSV events
5. Click Export Report → saves full audit to a .txt file

## Configuration (inside monitor.py)

|| Variable | Default | Description |

| SENSITIVE_EXTENSIONS | .pdf .docx .xlsx .csv .txt .pptx .db .sql .key .pem .env .json | File types flagged as sensitive |
| SUSPICIOUS_DESTINATIONS | ~/Downloads /tmp /media /mnt | Destinations that trigger unauthorized alerts |
| LOG_FILE | logs/file_transfer_log.json | Event log path |
| ALERT_FILE | logs/alerts.json | Alert log path |
| HASH_DB_FILE | logs/hash_db.json | Baseline hash database path |

You can also change sensitive extensions live in the GUI sidebar.
